                                             Section: Network Forensics – Mapping Techniques
-----------------------------------------------------------------------------------------------------------------------------------------

Mapping   | Line Types – Use distinct styles for wired, wireless, or virtual links.

-----------------------------------------------------------------------------------------------------------------------------------------
                                             Section: Network Forensics – Device & Interface Details
-----------------------------------------------------------------------------------------------------------------------------------------

          | Device Type – Identify whether each asset is a router, host, switch, etc.  
Details   | System Hostnames – Label each system with its hostname.  
          | Interface Names – Include all interfaces (e.g., eth0, eth1, wlan0).  
          | IP Address & CIDRs – Document IPs and subnet masks for each interface.  
          | TCP & UDP Ports – Note open or listening ports per device.  
          | MAC Address – Record hardware addresses for network interfaces.  
          | OS Type/Version – Specify operating system and version.  
          | Known Credentials – Log any discovered usernames/passwords for access.

-----------------------------------------------------------------------------------------------------------------------------------------
                                             Section: Network Recon Methodology – Tactical Guidance
-----------------------------------------------------------------------------------------------------------------------------------------

          | Suggested Techniques – Framework for conducting internal recon and mapping.  
Methodology|     - Start with passive discovery (logs, sniffers, ARP tables).  
            |     - Use active scanning (ping, Nmap, /dev/tcp) to identify hosts and services.  
            |     - Validate findings with banner grabbing and credential testing.  
            |     - Document all assets and relationships in diagrams.  
            |     - Use mapping tools to visualize and communicate findings.  
            |     - Continuously update based on new intel or changes in the environment.
