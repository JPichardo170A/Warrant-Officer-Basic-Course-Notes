# IPv4 Address Classes – Summary
---------------------------------
IPv4 originally defined five address classes (A–E), each serving different network sizes and purposes.
These are identified by the first few bits of the IP address.

Class A
- Range: 1.0.0.0 to 126.255.255.255
- CIDR Notation: /8
- Subnet Mask: 255.0.0.0
- Hosts per Network: ~16 million
- Use: Very large networks
- Binary Prefix: 0xxxxxxx
- Binary Range:
  - Start: 00000000.00000000.00000000.00000000 (0.0.0.0)
  - End:   01111111.11111111.11111111.11111111 (127.255.255.255)
- Example: 10.0.0.1 → 00001010.00000000.00000000.00000001

Class B
- Range: 128.0.0.0 to 191.255.255.255
- CIDR Notation: /16
- Subnet Mask: 255.255.0.0
- Hosts per Network: ~65,000
- Use: Medium-sized networks
- Binary Prefix: 10xxxxxx
- Binary Range:
  - Start: 10000000.00000000.00000000.00000000 (128.0.0.0)
  - End:   10111111.11111111.11111111.11111111 (191.255.255.255)
- Example: 172.16.0.1 → 10101100.00010000.00000000.00000001

Class C
- Range: 192.0.0.0 to 223.255.255.255
- CIDR Notation: /24
- Subnet Mask: 255.255.255.0
- Hosts per Network: 254
- Use: Small networks
- Binary Prefix: 110xxxxx
- Binary Range:
  - Start: 11000000.00000000.00000000.00000000 (192.0.0.0)
  - End:   11011111.11111111.11111111.11111111 (223.255.255.255)
- Example: 192.168.1.1 → 11000000.10101000.00000001.00000001

Class D (Multicast)
- Range: 224.0.0.0 to 239.255.255.255
- CIDR Notation: N/A
- Use: Multicast group communication
- Binary Prefix: 1110xxxx
- Binary Range:
  - Start: 11100000.00000000.00000000.00000000 (224.0.0.0)
  - End:   11101111.11111111.11111111.11111111 (239.255.255.255)
- Example: 224.0.0.1 → 11100000.00000000.00000000.00000001

Class E (Experimental)
- Range: 240.0.0.0 to 255.255.255.255
- CIDR Notation: N/A
- Use: Reserved for experimental purposes
- Binary Prefix: 1111xxxx
- Binary Range:
  - Start: 11110000.00000000.00000000.00000000 (240.0.0.0)
  - End:   11111111.11111111.11111111.11111111 (255.255.255.255)
- Example: 240.0.0.1 → 11110000.00000000.00000000.00000001

Note:
Modern networks use CIDR (Classless Inter-Domain Routing) instead of class-based addressing for more flexible subnetting.

# Reserved IPv4 Address Ranges
-------------------------------

1. Private IP Ranges (Not routable on the public internet)
   - 10.0.0.0 to 10.255.255.255         (Class A)
   - 172.16.0.0 to 172.31.255.255       (Class B)
   - 192.168.0.0 to 192.168.255.255     (Class C)
   - Purpose: Used for internal LANs; requires NAT for internet access
2. Loopback Addresses
   - 127.0.0.0 to 127.255.255.255
   - Purpose: Used for internal host communication (e.g., localhost testing)
3. Link-local Automatic Private IP Addressing (APIPA)
   - 169.254.0.0 to 169.254.255.255
   - Purpose: Auto-assigned when DHCP fails; used for local communication only
4. Multicast Range
   - 224.0.0.0 to 239.255.255.255
   - Purpose: Used for multicast traffic (e.g., streaming, routing protocols)
5. Broadcast Address
   - 255.255.255.255
   - Purpose: Sends traffic to all hosts on a local network
6. Experimental / Reserved for Future Use
   - 240.0.0.0 to 255.255.255.254
   - Purpose: Reserved for future or experimental use; not publicly routable
7. Network ID / Default Route
   - 0.0.0.0
   - Purpose: Represents "this network" or default route in routing tables

--------------------
DHCP D.O.R.A Process
--------------------

D – DHCP Discover
- The client sends a broadcast message to locate available DHCP servers.
- Destination IP: 255.255.255.255
- Destination MAC: ff:ff:ff:ff:ff:ff
- Purpose: "Who can give me an IP address?"

O – DHCP Offer
- DHCP servers respond with an available IP address and configuration details.
- Includes: IP address, subnet mask, default gateway, DNS servers
- Sent as a unicast to the client’s MAC address
- Purpose: "Here’s what I can offer you."

R – DHCP Request
- The client selects one offer and sends a request to that DHCP server.
- Includes the offered IP address
- May send requests to multiple servers but accepts only one
- Purpose: "I’d like to use this IP address."

A – DHCP Acknowledge (ACK)
- The DHCP server confirms the IP assignment and reserves it for the client.
- Sends a unicast ACK message with final configuration details
- Purpose: "You’re officially assigned this IP address."

Summary:
The D.O.R.A process ensures dynamic IP allocation and configuration 
for devices joining a network, enabling seamless connectivity without manual setup.

# TTL (Time To Live) – Platform-Based Defaults
----------------------------------------------

TTL is a field in the IP header that limits the lifespan of a packet. Each router hop decrements the TTL by 1.
If TTL reaches 0, the packet is discarded. This helps prevent routing loops.

Key Insight: Unless you capture the packet directly from the source host, 
             the TTL value will likely be lower than its original default due to hop reduction.

Typical TTL Ranges by Platform:
-------------------------------

- Linux: TTL defaults between 64 and 128  
  - Common observed range: 34–64
- Windows: TTL defaults to 128  
  - Common observed range: 98–128
- Cisco Devices: TTL defaults to 255  
  - Common observed range: 225–255

General Rule:
- Most internet destinations are reachable within ~30 hops, 
  so TTL values observed in packet captures are often 30 or more below their original default.
Use Case:
- TTL values can help identify the operating system of a remote host during network reconnaissance or troubleshooting.
--------------------------------------------------------------------------------------------------------------------------

----------------------------------
Traceroute (Linux) – Quick Summary
----------------------------------

Traceroute maps the path packets take to a destination by sending packets with increasing TTL values. 
Each router that drops a packet (TTL = 0) replies with an ICMP "Time Exceeded" message, revealing its IP and response time.

Basic Command:
  traceroute <destination>

Common Options:
- `-n` : Show numeric IPs only (skip DNS lookup)
- `-I` : Use ICMP Echo instead of default UDP
- `-T` : Use TCP SYN packets
- `-m <max_ttl>` : Set max number of hops
- `-p <port>` : Specify destination port (UDP/TCP)

Use Case:
Helps diagnose routing issues, latency, or unreachable nodes across networks.

# IPv6 Private Addressing and Class Structure
-----------------------------------------------

IPv6 does not use traditional address classes like IPv4 (A, B, C). Instead, it uses prefix-based allocation and address types defined by function.

Private Address Type:
- Unique Local Address (ULA)
  - Prefix: fc00::/7
    - Commonly used: fd00::/8 (locally assigned)
  - Purpose: Private addressing within an organization
  - Not routable on the public internet
  - Equivalent to IPv4 private ranges (e.g., 10.0.0.0/8)

Other Functional Address Types:
- Global Unicast
  - Prefix: 2000::/3
  - Routable on the public internet
  - Assigned by IANA to ISPs and organizations

- Link-Local
  - Prefix: fe80::/10
  - Used for communication between devices on the same link
  - Automatically assigned; not forwarded by routers

- Multicast
  - Prefix: ff00::/8
  - Used for one-to-many communication

- Loopback
  - Address: ::1
  - Used for internal host communication (like 127.0.0.1 in IPv4)

Summary:
IPv6 replaces class-based structure with functional address types. ULA (fd00::/8) serves as the private address space. 
Address types are defined by prefix, not by class.


## IPv6 Address Representation – Shortening Rules
----------------------------------------------

Full Address:
2001:0000:0000:0001:0000:0000:0000:0001

Shortening Steps:
1. Remove leading zeros in each hextet:
   → 2001:0:0:1:0:0:0:1

2. Replace one sequence of consecutive zero hextets with double colon (::):
   → 2001:0:0:1::1

Important Rules:
- Leading zeros in each hextet can be omitted.
- Consecutive zero hextets can be replaced with "::" only once per address.
- "::" represents one or more groups of 0000, but must not be used more than once to avoid ambiguity.

Examples:
- Full: 2001:0000:0000:0001:0000:0000:0000:0001  
- Shortened: 2001:0:0:1::1


## IPv6 Neighbor Discovery Protocol (NDP)
--------------------------------------

NDP is a core part of IPv6, replacing ARP and parts of DHCP. 
It enables devices to discover routers, resolve IP-to-MAC addresses, and maintain reachability information.

Message Types:
1. Router Solicitation (Type 133)
   - Sent by a host to request router information when joining a network.
   - Typically sent as a multicast to all routers (ff02::2).
   - Purpose: "Are there any routers out there?"

2. Router Advertisement (Type 134)
   - Sent by routers in response to solicitations or periodically.
   - Contains prefix info, default gateway, MTU, and flags for SLAAC or DHCPv6.
   - Purpose: "Here’s how to configure yourself on this network."

3. Neighbor Solicitation (Type 135)
   - Sent by a device to determine the MAC address of a neighbor (similar to ARP).
   - Also used to verify reachability of a neighbor.
   - Purpose: "Who owns this IPv6 address?"

4. Neighbor Advertisement (Type 136)
   - Sent in response to a Neighbor Solicitation.
   - Contains the MAC address of the responding device.
   - Purpose: "I own that address—here’s my MAC."

5. Redirect (Type 137)
   - Sent by a router to inform a host of a better next-hop for a destination.
   - Helps optimize routing paths within a local network.
   - Purpose: "Use this better route instead."

Summary:
NDP enables dynamic discovery and communication between IPv6 devices without manual configuration. 
It supports address resolution, router discovery, and efficient routing updates.

# Administrative Distance (AD) – Quick Reference
-----------------------------------------------
Definition:
Administrative Distance is a value used by routers to rank the trustworthiness of different routing sources. 
Lower AD values are preferred over higher ones when multiple routes to the same destination exist.

Default AD Values by Routing Source:
------------------------------------
0   - Connected (directly attached interface)
1   - Static route (manually configured)
5   - EIGRP summary route  (Enhanced Interior Gateway Routing Protocol)
20  - External BGP  (Border Gateway Protocol – external)
90  - EIGRP internal  (Enhanced Interior Gateway Routing Protocol)
100 - IGRP  (Interior Gateway Routing Protocol)
110 - OSPF  (Open Shortest Path First)
115 - IS-IS  (Intermediate System to Intermediate System)
120 - RIP  (Routing Information Protocol)
170 - Internal BGP  (Border Gateway Protocol – internal)
200 - ODR  (On-Demand Routing)
255 - Unknown or unreachable  (Route will not be used)

Key Notes:
- AD is only relevant when multiple routing protocols provide routes to the same destination.
- The route with the **lowest AD** is installed in the routing table.
- AD can be manually adjusted to influence route selection.

Example:
If both RIP (AD 120) and OSPF (AD 110) provide a route to 192.168.1.0/24, the router will prefer the OSPF route.

# Routing vs. Routed – Quick Reference
---------------------------------------

Routing:
- Refers to the process of selecting paths in a network along which to send data.
- Involves protocols that help routers exchange information and build routing tables.
- Focuses on **how** data gets from one point to another.
- Examples of routing protocols:
  - RIP (Routing Information Protocol)
  - OSPF (Open Shortest Path First)
  - EIGRP (Enhanced Interior Gateway Routing Protocol)
  - BGP (Border Gateway Protocol)

Routed:
- Refers to the actual data being transported across the network.
- Involves protocols that carry user traffic from source to destination.
- Focuses on **what** is being moved across the network.
- Examples of routed protocols:
  - IP (Internet Protocol)
  - IPv6 (Internet Protocol version 6)
  - AppleTalk
  - IPX (Internetwork Packet Exchange)

Summary:
- **Routing** is the decision-making process.
- **Routed** is the data-carrying mechanism.
- Routing protocols build the map; routed protocols use it to deliver the message.

# Autonomous Systems (AS)
---------------------------

Definition:
An Autonomous System is a collection of connected IP routing prefixes under the control of one or more network operators, acting as a single administrative domain. It presents a unified routing policy to the Internet.

Key Characteristics:
- Identified by a unique 16-bit or 32-bit AS number (ASN)
- Used in BGP (Border Gateway Protocol) to define routing boundaries

Examples:
AS109    CISCO-EU-109 – Cisco Systems Global ASN  
AS193    FORD-ASN – Lockheed Martin Western Development Labs  
AS721    DoD Network Information Center Network  
AS3598   MICROSOFT-CORP-AS – Microsoft Corporation  
AS15169  GOOGLE – Google Inc.

Routing Protocol Concepts
-------------------------

Distance Vector Protocols:
- Share entire routing tables with neighbors at regular intervals
- Use hop count as the primary metric
- Routers do not have full network topology awareness
- Examples:
  - RIP (Routing Information Protocol)
  - IGRP (Interior Gateway Routing Protocol)
- Pros: Simple to configure
- Cons: Slower convergence, risk of routing loops

Link-State Protocols:
- Share only topology changes (not full routing tables)
- Each router builds a complete map of the network
- Use Dijkstra’s algorithm to calculate shortest paths
- Examples:
  - OSPF (Open Shortest Path First)
  - IS-IS (Intermediate System to Intermediate System)

LSA – Link-State Advertisement:
- A type of message used in link-state protocols (especially OSPF)
- Contains information about a router’s interfaces, neighbors, and network links
- Flooded throughout the area to ensure all routers have synchronized topology data
- Enables fast convergence and accurate path selection

Summary:
- Autonomous Systems define routing domains in BGP
- Distance Vector protocols rely on neighbor updates and hop counts
- Link-State protocols use LSAs to build a full network map for efficient routing

# First Hop Redundancy Protocols (FHRP)
---------------------------------------

1. HSRP – Hot Standby Router Protocol
   - Proprietary to Cisco
   - Provides gateway redundancy by creating a virtual IP address shared between routers
   - One router is active, others are in standby
   - Clients use the virtual IP as their default gateway
   - Only the active router forwards traffic

2. VRRP – Virtual Router Redundancy Protocol
   - Open standard (RFC 5798)
   - Similar to HSRP but supports multiple vendors
   - One router is elected as master; others are backups
   - Clients use a virtual IP; only the master forwards traffic

3. GLBP – Gateway Load Balancing Protocol
   - Cisco proprietary
   - Provides both redundancy and load balancing
   - Multiple routers share the forwarding load
   - Clients receive different virtual MAC addresses for the same virtual IP
   - One router acts as the Active Virtual Gateway (AVG); others are Active Virtual Forwarders (AVFs)

Comparison Summary:
-------------------

| Protocol | Type        | Vendor Support | Redundancy | Load Balancing |
|----------|-------------|----------------|------------|----------------|
| HSRP     | Proprietary | Cisco only     | Yes        | No             |
| VRRP     | Open        | Multi-vendor   | Yes        | No             |
| GLBP     | Proprietary | Cisco only     | Yes        | Yes            |

Use Case:
- HSRP and VRRP are ideal for simple failover
- GLBP is preferred when both redundancy and traffic distribution are needed
