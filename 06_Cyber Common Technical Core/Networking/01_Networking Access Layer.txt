##Importance of the OSI Model

The OSI model describes networking functions as a set of layered, modular components, each responsible for particular functions. The model is theoretical in nature as certain networking protocols don't always fall nicely within a specific layer. It merely provides a framework for breaking down complex networking components in a way that can be more easily understood. Without the model's structure to help frame the conversation related to protocol interaction and other functions, discussion of networking topics would be nearly impossible.

## LAN Technologies: Benefits and Hindrances
-----------------------------------------

LAN (Local Area Network) technologies are used to connect devices within a limited geographical area, such as a home, office, or campus. These technologies are mostly governed by IEEE standards.

-----------------------------------------------------------------------------------------------

                                                                         - Easy to install, maintain, and troubleshoot
                                                                         - Fast and consistent data transfer speed

                                                                         - Allows multiple protocols to use the same network medium
                                                                         - Adds extra layers of logic and control to the network

## ARP Types Summary
----------------------
ARP & RARP Operation Codes
----------------------------------------------------------------------------
1 = ARP Request
    - A device asks: "Who has this IP address? Tell me your MAC address."
2 = ARP Reply
    - The device with that IP responds: "Here’s my MAC address."
3 = RARP Request
    - A device asks: "I know my MAC address—can someone tell me my IP?"
4 = RARP Reply
    - A server responds with the IP address for that MAC.
----------------------------------------------------------------------------

1. ARP (Address Resolution Protocol): A request/response mechanism to resolve the destination Layer 2 (MAC) address when only the destination Layer 3 (IPv4) address is known
   - Operation Codes: OP 1 and OP 2
   - Used to map IP addresses to MAC addresses within a local network.

2. RARP (Reverse ARP): Resolves a device’s own IP address using its known MAC address.
   - Operation Codes: OP 3 and OP 4
   - Used to map MAC addresses back to IP addresses (mostly obsolete now).

3. Proxy ARP: A router (or gateway) responds to ARP requests on behalf of devices on other networks.
   - Operation Code: OP 2
   - A router answers ARP requests on behalf of another device, allowing communication across subnets.

4. Gratuitous ARP: An unsolicited ARP reply announcing a device’s own IP-to-MAC mapping.
   - Operation Code: OP 2
   - A device sends an ARP request for its own IP to check for conflicts or update other devices' ARP tables.

1. VLAN Trunking Protocol (VTP)
--------------------------------
VTP is a Cisco proprietary protocol used to manage VLAN configurations across multiple switches. It allows administrators to make changes on one switch and automatically propagate those changes to others in the same VTP domain—ensuring consistency and reducing errors.

Key Concepts:

- VTP Domain:
  A group of switches sharing the same VTP configuration and VLAN information.
  All switches in the domain synchronize VLAN databases.
  Each switch must be configured with the same domain name.

- VTP Modes:

  | Mode        | Description                                                                 | Default |
  |-------------|------------------------------------------------------------------------------|---------|
  | Server      | Can create, modify, delete VLANs. Shares VLAN info with other switches.     | Yes     |
  | Client      | Receives VLAN info from servers. Cannot create, modify, or delete VLANs.     | No      |
  | Transparent | Forwards VTP advertisements but does not apply them. Maintains own VLANs.    | No      |
  | Off         | Disables VTP entirely. Not a standard mode but used to indicate inactivity.  | No      |

- VTP Advertisements:

  | Type                   | Purpose                                                            |
  |------------------------|--------------------------------------------------------------------|
  | Summary Advertisements | Share domain name, revision number, and VLAN info.                |
  | Subset Advertisements  | Carry VLAN configuration changes.                                 |
  | Advertisement Requests | Ask other switches for current VLAN info.                         |

- VTP Revision Number:
  Included in every VTP advertisement.
  Increments with each VLAN configuration change.
  Higher numbers indicate more recent configurations.
  Used to determine whether switches should update their VLAN database.

- VTP Versions:
  - Version 1
  - Version 2
  - Version 3

Each version adds enhancements in VLAN support, security, and extended features.

2. Dynamic Trunking Protocol (DTP)
----------------------------------
DTP is a Cisco proprietary protocol used to negotiate trunk links between switches. It allows switches to automatically form trunk links by:

- Negotiating trunking encapsulation (e.g., IEEE 802.1Q)
- Determining whether a link should operate as a trunk or access port

Benefits:
- Simplifies network configuration
- Automates trunk formation between compatible switches

Risks:
- Introduces vulnerabilities that can be exploited in various types of attacks
- May allow unauthorized trunk formation if not properly secured

Best Practice:
- Disable DTP on access ports using `switchport nonegotiate` to reduce attack surface

Discovery Protocols Summary
---------------------------
- CDP (Cisco Discovery Protocol)  
  Cisco proprietary protocol that allows Cisco devices to advertise themselves to directly connected Cisco neighbors.  
  Shares info like device ID, IP address, port, platform, and capabilities.  
  Used for network mapping and troubleshooting in Cisco-only environments.

- FDP (Foundry Discovery Protocol)
  Operates at the Data Link Layer (Layer 2).
  Proprietary protocol by Foundry Networks (now Brocade).  
  Works similarly to CDP, enabling Foundry devices to discover and share details with neighbors.  
  Used for device identification and management in Foundry-based networks.

- LLDP (Link Layer Discovery Protocol)
  Operates at Layer 2, independent of IP.
  IEEE 802.1AB standard protocol—vendor-neutral.  
  Allows devices from different manufacturers to advertise identity, capabilities, and management info.  
  Ideal for multi-vendor environments; supports cross-platform discovery.

# Spanning Tree Protocol (STP) – Root Decision Process
----------------------------------------------------

1. Elect Root Bridge
   - The switch with the lowest Bridge ID (priority + MAC address) becomes the Root Bridge.
   - All path calculations are made relative to this switch.

2. Identify Root Ports (on Non-Root Bridges)
   - Each non-root switch selects one port with the lowest cost path to the Root Bridge.
   - This port becomes the **Root Port** and is used to reach the Root Bridge.

3. Identify Designated Ports (per Segment)
   - For each network segment, the switch with the lowest path cost to the Root Bridge has its port elected as the **Designated Port**.
   - This port forwards traffic toward the Root Bridge for that segment.

4. Set Alternate Ports to Blocking State
   - Any port not selected as a Root or Designated Port is placed in **Blocking** state.
   - This prevents loops by disabling redundant paths.

## STP Variants – Functional Summary

- 802.1D (STP)  
  Prevents Layer 2 loops by creating a single spanning tree.  
  Slow convergence (~30–50 seconds).

- PVST+ (Per VLAN STP Plus)  
  Cisco proprietary. Builds a separate spanning tree for each VLAN.  
  Allows VLAN-specific path control.

- 802.1w (RSTP – Rapid STP)  
  Faster version of STP with quicker convergence (<10 seconds).  
  Backward compatible with 802.1D.

- RPVST+ (Rapid PVST Plus)  
  Cisco proprietary. Combines RSTP speed with per-VLAN control.  
  Rapid convergence for each VLAN.

- 802.1s (MST – Multiple STP)  
  Maps multiple VLANs to a single spanning tree instance.  
  Reduces overhead and improves scalability in large networks.
