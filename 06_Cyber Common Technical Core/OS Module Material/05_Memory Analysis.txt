=======================================
 14_WINDOWS MEMORY ANALYSIS
=======================================

🔹 Basic Volatility Command Syntax  
    PS C:\Users\andy.dwyer\Desktop\Memory_Analysis> .\volatility_2.6_win64_standalone.exe -h | findstr /i con  

🔹 Example: Profile Analysis + Running Processes  
    .\volatility_2.6_win64_standalone.exe -f ".\cridex.vmem" --profile=WinXPSP2x86 pslist  


1. What Volatility plugin will dump a process to an executable file sample?  
    -> procdump  

2. What Volatility plugin will extract command history by scanning for _COMMAND_HISTORY?  
    -> cmdscan  

3. What Volatility plugin will show driver objects?  
    -> driverscan  

4. What plugin do you run to find which memory profile to use with a memory image?  
    -> imageinfo  

5. What switch/argument will list all plugins for Volatility?  
    -> --help  

6. In terms of Volatile Data, what locations are the MOST volatile?  
   Flag format: location,location  
    -> registers,cache  

7. What is the 12th plugin listed in the Volatility help menu?  
    -> cmdscan  

8. What profile do you use in conjunction with this memory image?  
   (File: 0zapftis.vmem)  

    -> Step 1: Run the imageinfo plugin.  
               Command: .\volatility_2.6_win64_standalone.exe -f .\0zapftis.vmem imageinfo  

    -> Step 2: Review "Suggested Profile(s)".  
               First suggested profile = most accurate  

    -> Flag: WinXPSP2x86  

9. What command did the attacker type to check the status of the malware?  

    -> Step 1: Run the `cmdscan` plugin.  
               Command: .\volatility_2.6_win64_standalone.exe -f .\0zapftis.vmem --profile=WinXPSP2x86 cmdscan  

    -> Step 2: Review extracted command history.  
               Found: sc query malware  

    -> Flag: sc query malware  

10. What are the last 7 digits of the memory offset for the driver used by the malware?  

    -> Step 1: Run the `driverscan` plugin.  
               Command: .\volatility_2.6_win64_standalone.exe -f .\0zapftis.vmem --profile=WinXPSP2x86 driverscan  

    -> Step 2: Look for driver named "malware".  
               Example entry:  
               0x0000000001a498b8        3   0  0xf9eb4000   0x1500   malware   malware   \Driver\malware  

    -> Flag: 1a498b8  

11. The process running under PID 544 seems malicious.  
    What is the md5hash of the executable?  

    -> Step 1: Create directory to dump executable.  
               Command: mkdir .\dump  

    -> Step 2: Dump PID 544 process executable.  
               Command: .\volatility_2.6_win64_standalone.exe -f .\0zapftis.vmem --profile=WinXPSP2x86 procdump -p 544 -D .\dump  

    -> Step 3: Calculate MD5 hash in PowerShell.  
               Command: Get-FileHash .\dump\executable.544.exe -Algorithm MD5  

    -> Flag: 6CEE14703054E226E87A963372F767AA  

12. What remote IP and port did the system connect to?  
    Flag format: ip:port  

    -> Step 1: Run the `connscan` plugin.  
               Command: .\volatility_2.6_win64_standalone.exe -f .\0zapftis.vmem --profile=WinXPSP2x86 connscan  

    -> Step 2: Review connections.  
               Example:  
               Local Address: 0.0.0.0:1026  
               Remote Address: 172.16.98.1:6666  
               PID: 1956  

    -> Flag: 172.16.98.1:6666
