# ☁️ Cloud Security & Governance

*Module 04 — Cloud Foundations | Lesson 00*

---

## 🎯 Learning Objectives

- Understand data protection and regulatory compliance in the cloud
- Identify key cloud security concepts (IAM, encryption, network security, threat detection, Zero Trust)
- Compare On-Prem, Cloud, and Hybrid deployment models
- Understand the Shared Responsibility Model for AWS and Azure

---

## 🧱 Introduction to Cloud Security & Governance

Cloud security and governance encompasses four critical areas:

| Area | Description |
|------|-------------|
| **Data Protection** | Safeguarding data at rest and in transit |
| **Regulatory Compliance** | Meeting industry and government standards |
| **Risk Mitigation** | Reducing exposure to threats and vulnerabilities |
| **Efficiency** | Streamlining security operations through automation |

---

## 🧱 Key Concepts of Cloud Security & Compliance

### Cloud Security

| Concept | Description |
|---------|-------------|
| **IAM** | Identity and Access Management |
| **Encryption** | Protecting data at rest and in transit |
| **Network Security** | Firewalls, VPCs, security groups |
| **Threat Detection** | Monitoring and alerting on suspicious activity |
| **Zero Trust** | Never trust, always verify |

### Cloud Compliance

Ensuring cloud operations meet regulatory requirements and organizational policies.

---

## 🧱 On-Prem vs. Cloud vs. Hybrid Models

| Model | Description |
|-------|-------------|
| **On-Premises** | All infrastructure owned and managed locally |
| **Cloud** | Infrastructure hosted by cloud provider (AWS, Azure, GCP) |
| **Hybrid** | Combination of on-prem and cloud resources |

---

## 🧱 Shared Responsibility Model (AWS & Azure)

| Responsibility | Owner |
|---------------|-------|
| **AWS Responsibility** | Security *of* the cloud — infrastructure, hardware, networking, managed services |
| **Customer Responsibility** | Security *in* the cloud — data, identity, application, OS patching, firewall configuration |

> 💡 **Key Takeaway:** The shared responsibility model means the cloud provider secures the infrastructure, but **you** are responsible for securing your data, access, and configurations within that infrastructure.

---

## ✅ Summary

- Cloud security rests on IAM, encryption, network security, threat detection, and Zero Trust
- Three deployment models: on-prem, cloud, and hybrid
- The Shared Responsibility Model divides security duties between provider and customer
- Compliance and governance are non-negotiable in cloud operations
